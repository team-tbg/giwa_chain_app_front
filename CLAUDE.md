# CLAUDE.md — 나드리(Naduri) 프로젝트 가드레일

이 파일은 매 세션 컨텍스트에 자동 로드된다. 아래 **불변 규칙**을 어기는 코드·카피·설계 제안을 하지 마라. 애매하면 규칙 편에서 보수적으로 판단하고, 사용자에게 확인하라.

## 프로덕트 한 줄
나드리 = 걷기(만보기)로 크립토 무경험자를 입장시켜, 걸어서 번 **리워드**로 디파이를 처음 맛보게 하고, 그 신뢰로 **원금 예치(→ 기와체인 TVL)** 까지 데려오는 **안드로이드 우선(React Native/Expo) 모바일 앱**. 기와체인(OP Stack EVM L2) + 업비트 정산 레일 위.

## 불변 규칙 (절대 위반 금지)
1. **유저를 속이지 않는다.** 가짜 잔고, 숨은 차감, 오해 유도 UI 금지. 신뢰가 유일한 해자.
2. **출금(받기)은 항상 원탭·진짜 열림.** 강요·마찰·"정말 나가실 건가요?"·인질형("출금하려면 입금") 금지. 3탭 이내.
3. **확정 수익률/원금 보장 약속 금지.** 미래 가치·이자는 항상 가정형 + "달라질 수 있어요". (규제: 유사수신·투자권유 회피)
   - 벤치마크 unifi.me가 "8%/3% 이율"을 노출하더라도 **우리는 확정 % 표기하지 않는다**(퍼센트 대신 "매일 붙는 금액").
4. **본인 원금 예치 화면엔 원금 비보장·손실 가능 고지 필수.** "원금 걱정 없어요"는 걸어서 번 리워드(하우스 머니)에 한정.
5. **체인은 유저 눈앞에서 숨긴다.** 지갑·가스·시드문구·스왑·코인명·퍼센트를 UI에 노출하지 않는다(임베디드 월렛/AA 뒤로).
6. **idle 잔고·위치·의도 데이터는 되비추는 거울 용도만.** 서버 축적·판매 금지.
7. **인앱 크레딧/포인트는 폐쇄형(자사 전용).** 제3자 상품권·현금·코인 구매 수단으로 확장 금지(선불전자지급수단·특금법 회피).
8. **-EV 갓챠·사행성 뽑기 금지.**
9. **토큰 발행은 기본이 아니다.** 요청 없이 토크노믹스·플랫폼 토큰 제안 금지.

## 지금 단계 (테스트넷 / 파일럿 / GASOK 2차)
- 만든다: 로그인, 걷기·적립, 오프체인 포인트 원장, 체험 예치 UX, 이자 시뮬레이션(우리 재원), 출금 모멘트 전환, 스마트컨트랙트 "설계".
- 안 만든다: 실제 온체인 풀, 업비트 계정 연동(파트너십 후), 실데이터 idle 개인화, 광고 SDK, 토큰.
- 시연 원칙: **실제 구현이라 생각하고 진행** — 화면에 "목업/데모/다음 단계" 표기 노출 금지.
- 유일 검증 지표: 리워드 체험 → **첫 진짜 원금(₩1,000이라도) 전환율**.

## 톤 & 타깃
- 주 고객: 블록체인 용어를 모르는 일반인(40~50대 포함). 큰 글씨·명확한 대비·쉬운 일상어.
- 적립 명칭은 **"리워드"** (과거 "용돈"은 폐기). 크립토 용어 금지.

## 기술
- FE: React Native + Expo(SDK 57), TypeScript. 네비: React Navigation. 만보기: expo-sensors.
- 테마: 라이트 · 핀테크 블루. 토큰은 `src/theme/theme.ts`.
- 변경 후 `npx tsc --noEmit` 로 타입 확인.
