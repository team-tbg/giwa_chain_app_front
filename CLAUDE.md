# CLAUDE.md — 나드리(Naduri) 프로젝트 가드레일

이 파일은 매 세션 컨텍스트에 자동 로드된다. **디자인·제품의 기준(source of truth)은 `docs/prototype/naduri_v10.html`(v10 프로토타입)** 이다. 아래 규칙과 v10을 어기는 코드·카피·설계 제안을 하지 마라. 애매하면 규칙 편에서 보수적으로 판단하고, 사용자에게 확인하라.

## 프로덕트 한 줄
나드리 = 걷기(만보기)로 크립토 무경험자를 입장시켜, 걸어서 번 **포인트**로 디파이 이자를 처음 맛보게 하고, 그 신뢰로 **원화 예치(→ 기와체인 TVL)** 와 **비트코인·금 소액 매수**까지 데려오는 **안드로이드 우선(React Native/Expo) 모바일 앱**. 기와체인(GIWA, OP Stack EVM L2) + 업비트 정산 레일 위.

## 화폐 모델 (v10 기준)
- **포인트(P)** — 걸어서 버는 적립 화폐. 걸음·출석·퀴즈·게임·영상·보너스·룰렛·뽑기·설문·초대로 쌓인다.
- **원화 자산(원)** — 진짜 돈. 포인트샵에서 포인트를 현금으로 바꾸거나 직접 입금하면 쌓인다. 내 계좌로 출금(영업일 1~3일).
- **비트코인 / 금(PAXG)** — 포인트샵에서 포인트로 매수, 내 지갑에 보관, 원화로 되팔기 가능. 업비트 오라클 시세.
- **이자받기(저금)** — 포인트도 원화도 저금하면 GIWA 체인 디파이에서 이자가 붙는다(매 초 증가). 언제든 꺼낸다.

## 규제 (파일럿 단계 방침)
- **디파이 제품이므로, 파일럿/데모 단계에서는 규제(특금법·선불전자지급수단·유사수신·투자권유 등)를 고려하지 않는다.** 포인트↔현금·코인 전환, 지갑·코인·체인 노출을 규제 이유로 막지 않는다.
- 아래 유저 보호 규칙(1~9)은 규제가 아니라 **신뢰·정직·UX** 근거로 지킨다. 신뢰가 유일한 해자다.

## 불변 규칙 (유저 보호 — 절대 위반 금지)
1. **유저를 속이지 않는다.** 가짜 잔고, 숨은 차감, 오해 유도 UI 금지.
2. **출금(받기)은 항상 원탭·진짜 열림.** 강요·마찰·"정말 나가실 건가요?"·인질형("출금하려면 입금") 금지. 3탭 이내.
3. **확정 수익률/원금 보장 약속 금지.** 미래 가치·이자는 항상 가정형 + "시장 상황에 따라 달라질 수 있어요". 퍼센트 대신 **붙는 금액**으로 보여준다. (신뢰·정직 근거)
4. **손실 가능성을 감언이설로 숨기지 않는다.** 값이 오르내리는 자산(비트코인·금)엔 "산 금액보다 적어질 수 있어요"를 정직하게 고지(v10 구매 시점처럼). "원금 걱정 없어요"는 포인트(걸어서 번 하우스 머니) 저금에 한정. ← 없는 기능을 넣지 말라는 뜻이 아니라, 원금 잃을 수 있는데 없는 척하지 말라는 뜻.
5. **체인 복잡성은 대신 처리한다(임베디드 월렛/AA).** 가스·네트워크 선택·스왑 절차를 유저가 하게 만들지 않는다. 단, **지갑주소·비트코인·금·GIWA 체인은 내 지갑·포인트샵·이자받기 맥락에서 드러낸다(v10).** 시드문구·개인키는 "고급 설정"에만 둔다.
6. **idle 잔고·위치·의도 데이터는 되비추는 거울 용도만.** 서버 축적·판매 금지.
7. **포인트는 걸어서 버는 적립 화폐이며, 포인트샵에서 현금·비트코인·금·쿠폰으로 전환할 수 있다(v10).**
8. **-EV 갓챠·사행성 뽑기 금지.** 뽑기 경품 기대값이 참가비 이상이 되도록 설계한다.
9. **토큰 발행은 기본이 아니다.** 요청 없이 토크노믹스·플랫폼 토큰 제안 금지.

## 앱 구조 (v10 — 하단 6탭)
- **홈** — 걸음 링, 걸어서 받을 포인트, 오늘 모은 포인트(일일 한도 200P), 자산 스트립, 퀵 메뉴, 불리기 배너.
- **혜택** — 출석체크 · 오늘의 보너스(6h, ~300P) · 영상 보고 받기 · 게임하고 받기 · 나드리 뽑기 · 행운 룰렛 · OX 금융퀴즈 · 설문 · 친구 초대.
- **이자받기** — 포인트 저금 / 원화 저금(입금·출금) 두 트랙. GIWA 디파이 이자 시뮬레이션.
- **포인트샵** — 베스트 상품 · 카테고리 · 실시간 후기. 비트코인/금, 현금 전환, 쿠폰/기프티콘.
- **내 지갑** — 전체 자산(포인트/원화/BTC/금) 비중, 쿠폰함, 보유 자산별 출금·전환, 지갑주소(이름+주소+GIWA)·고급설정, 최근 내역.
- **내정보** — 프로필/닉네임, 초대, 알림, 활동 기록, 배지, 고객지원, 로그아웃.

일일 적립 한도: **200P**(걸음·출석·퀴즈·게임·영상). 보너스·룰렛·설문·친구 초대는 한도 제외.

## 지금 단계 (테스트넷 / 파일럿 / GASOK 2차)
- 만든다: 로그인, 걷기·적립, 오프체인 포인트 원장, 혜택 미니게임/적립, 포인트→현금/자산 전환 UX, 이자 시뮬레이션(우리 재원), 출금 모멘트 전환, 내 지갑·포인트샵, 스마트컨트랙트 "설계".
- 안 만든다: 실제 온체인 풀·실거래, 업비트 계정 연동(파트너십 후), 실데이터 idle 개인화, 광고 SDK, 플랫폼 토큰.
- 시연 원칙: **실제 구현이라 생각하고 진행** — 화면에 "목업/데모/다음 단계" 표기 노출 금지.
- 유일 검증 지표: 포인트 체험 → **첫 진짜 원금(₩1,000이라도) 전환율**.

## 톤 & 타깃
- 주 고객: 블록체인 용어를 모르는 일반인(40~50대 포함). 큰 글씨·명확한 대비·쉬운 일상어.
- 적립 화폐 명칭은 **"포인트(P)"** (v10). 어려운 크립토 절차 용어(가스·시드·스왑·네트워크)는 UI 전면 노출 금지. 비트코인·금·지갑주소는 맥락에서 노출 허용.

## 기술
- FE: React Native + Expo(SDK 57), TypeScript. 네비: React Navigation. 링/그래픽: react-native-svg. 위치·날씨: expo-location + Open-Meteo.
- 걸음수: Android Health Connect(`react-native-health-connect`, 백그라운드 포함) → 가속도계(expo-sensors) 폴백. 프론트 관리·DB 미저장.
- 로그인: 네이티브 `@react-native-google-signin`, 웹 expo-auth-session. (카카오=가상자산 제한, 토스=심사로 제외. 구글만.)
- 테마: 라이트 · 핀테크 블루 (v10 팔레트: blue `#2F6BF3`, grow `#12B76A`, gold `#FDB022`, 앱 배경 `#F5F7FB`). 토큰은 `src/theme/theme.ts`.
- 변경 후 `npx tsc --noEmit` 로 타입 확인.
