# Naduri — 내가 준비해야 할 것 (체크리스트)

> 최종 업데이트: 2026-07-22 · 대상: 프로덕트 오너 / 팀
> 목적: 앱을 **목업 → 실제 동작**으로 올리려면 코드 밖에서 준비해야 하는 것들(계정·키·파트너십·결정)을 한곳에.
> 코드/연동은 개발이 처리하고, 이 문서 항목은 **팀이 발급·계약·결정**해야 진행된다.

## 지금 상태 한 줄
앱(FE)은 v10 프로토타입대로 6탭이 **로컬 목업으로 동작**한다. 아직 **구현 안 된 기능은 화면에서 "준비 중" 안내**로 막아뒀다(가짜 성공 없음). 아래를 준비하면 하나씩 실제로 켜진다.

## 1. 소셜 로그인 🔴 가장 급함
로그인이 되어야 그 뒤 전부가 열린다. 코드는 이미 붙어 있고 **키만 넣으면 동작**한다.

### 1-1. 구글 로그인
- [ ] Google Cloud Console 프로젝트 생성
- [ ] **OAuth 동의 화면** 구성 (앱 이름 "나드리", 지원 이메일, 스코프 `openid`·`email`·`profile`)
- [ ] **OAuth 클라이언트 ID 3종** 발급
  - [ ] 웹 (Web) — Expo 웹 + id_token 검증 audience용
  - [ ] Android — 패키지명 **`io.giwa.naduri`** + 앱 서명 **SHA-1 지문** (8번 서명키에서 나옴)
  - [ ] iOS — 번들 ID (iOS 진행 시)
- [ ] 발급한 ID를 환경변수/`app.json`에 주입 (→ 10번 표)

### 1-2. 2차 소셜 로그인 (현재 보류 — 구글만 확정)
- **토스**: 심사 문턱 높음 → 제외.
- **카카오**: 사이트 운영정책 제3조 **"가상자산 관련 서비스(거래·보관·예치)"가 제한 카테고리** → 나드리(BTC/금 매수·보관, 예치)가 해당. 검수/출시 거절 위험 커서 **제외**.
- **네이버**: 명시적 가상자산 금지 조항은 안 보였으나(검수는 개인정보 수집 항목 중심), 확정 안 함 → **보류**.
- 필요 시 나중에 추가(구글 코드 구조 재사용). 지금은 **구글 단독**.

## 3. 입금 결제(PG) 🟡
입금 화면의 결제수단(토스/카카오페이/네이버페이/카드/계좌이체)을 실제로 쓰려면:
- [ ] PG사 계약 (토스페이먼츠 등) + 상점 키/시크릿
- [ ] 각 간편결제(카카오페이·네이버페이) 가맹 신청
- [ ] 결제 승인 결과를 **백엔드가 검증 후** 원화 자산 반영 (FE는 결과만)
> 파일럿/데모는 `method: "simulated"`로 실제 승인 없이 흐름만 검증 가능.

## 4. 자산·정산 (업비트/두나무 파트너십) 🟡
비트코인·금(PAXG) 매수/매도, 원화 출금, GIWA 예치가 실제로 돌려면:
- [ ] **업비트/두나무 파트너십** (라이선스 VASP가 커스터디·환전·정산 처리)
- [ ] **오라클 시세** 피드 (현재 고정값 `BTC 158,400,000 / 금 4,620,000`)
- [ ] 원화 **출금 정산** 레일 (영업일 1~3일) + 출금 계좌 인증(1원 인증 등)
- [ ] GIWA 체인 예치/이자 발생 컨트랙트 (지금은 "설계"만, 배포는 파트너십 후)

## 10. 환경변수 요약 (개발에 넘길 값)
발급받으면 아래 키로 주입한다. `EXPO_PUBLIC_*` 환경변수 또는 [app.json](../app.json) `extra` 둘 다 가능(환경변수 우선).

| 키 (env) | app.json extra | 용도 | 준비처 |
|---|---|---|---|
| `EXPO_PUBLIC_API_BASE_URL` | `apiBaseUrl` | 백엔드 주소 | 2번 |
| `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID` | `googleWebClientId` | 구글 로그인(웹/검증) | 1-1 |
| `EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID` | `googleAndroidClientId` | 구글 로그인(안드로이드) | 1-1 |
| `EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID` | `googleIosClientId` | 구글 로그인(iOS) | 1-1 |

> 참고: 안드로이드 패키지명 `io.giwa.naduri`, 앱 스킴 `naduri` (app.json).

## 우선순위 요약
1. **지금 당장**: 1(소셜 로그인 키) + 2(백엔드 + API URL) → 로그인부터 실제로 켜짐.
2. **그 다음**: 3(결제) · 5(이자 재원) → 입금·저금 루프 실제화.
3. **파트너십/제휴 후**: 4(업비트) · 6(쿠폰) · 7(배포).
4. **정식 출시 전**: 8(법무).

> 준비된 항목이 생기면 개발이 해당 기능의 "준비 중" 알림을 실제 동작으로 교체한다.
