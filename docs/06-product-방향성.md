# 프로덕트 방향성 (나드리 / Naduri)

> 최종 업데이트: 2026-07-22

## 한 줄 정의

**걷기(만보기) 등 일상 행동으로 포인트를 모으고, 그 포인트를 "불려서"(DeFi 유동성풀) 늘리는 경험을 통해, 블록체인 용어를 모르는 일반인에게 탈중앙화 금융을 자연스럽게 알리는 안드로이드 앱.**

## 타깃 & 톤

- **주 고객**: 블록체인/크립토 용어를 잘 모르는 일반인 (프로토타입 기준 40~50대 대중 톤)
- **원칙**: 화면에서 크립토·지갑·DeFi 전문 용어를 걷어내고, "리워드 / 불리기 / 이자 / 출금" 같은 일상어로 표현
- **테마**: 화이트 배경 · **핀테크 블루** 강조(리워드/포지티브는 그린). 토큰/카피 규칙은 [09-design-theme](./09-design-theme.md)
- **적립 명칭**: "리워드" (기존 "용돈"은 저렴한 인상이라 폐기)

## 플랫폼

- **모바일 앱 MVP**, **안드로이드 우선** (맥 미보유로 iOS 후순위)
- 백엔드/스마트컨트랙트는 GIWA 체인(OP Stack EVM L2) 위에 구성 → 자세한 건 [02-giwa-체인](./02-giwa-체인.md)

## 비즈니스 모델

1. **광고 수익**
2. **적립 → 활용 루프**
   - 만보기 등으로 **포인트 적립**
   - 포인트를 **출금**(현금화) 하거나 **불리기**(연이자 %로 예치)
   - "불리기"는 두 단계:
     - (a) 걸어서 번 포인트를 예치 = *하우스 머니*, 심리적 부담 낮음
     - (b) **내 원금을 추가 납입(deposit) = 유동성풀 투자** ← 실제 수익 모델의 핵심 전환
3. **적립 트리거는 만보기에 한정하지 않음**: 화면 잠금 해제, 알림 해제 등 확장 가능

## MVP 우선 개발 범위 (지금 당장)

- [ ] **로그인** (인증)
- [ ] **만보기 기능** (걸음 수 측정/집계)
- [ ] **포인트 조회** (적립 포인트 확인)
- [ ] (그 다음) 불리기 = 예치/출금 UI, 유동성풀 연동

| # | 화면 | 역할 |
|---|------|------|
| 0 | 온보딩 | "걸으면 쌓이는 리워드" 진입 |
| L | 로그인 | SNS 간편 로그인(구글·토스) — 앱에만 있는 단계 |

## ⚠️ 반드시 지킬 것 — 정직성 / 법적 리스크

사용자가 명시적으로 강조: **"사기는 치면 안 됨"** (예: "원금 걱정이 없어요")

프로토타입에는 다음과 같은 문구가 있음:
- "원금 걱정이 없어요"
- "은행에 그냥 두는 것보다 조금 더 받을 수 있어요"
- "10년 뒤 예상 약 486만원"

이 문구들은 **걸어서 번 포인트(하우스 머니)** 예치에는 어느 정도 성립할 수 있으나,
**사용자 본인 원금을 유동성풀에 예치(4→5번 전환)** 하는 순간부터는 사실과 다를 수 있어 위험함:

- 유동성풀은 **비영구적 손실(impermanent loss)**, **스마트컨트랙트 리스크**, **디페그/청산 리스크**가 존재 → "원금 걱정 없음"은 **거짓**이 될 수 있음
- 확정 수익률/원금 보장을 약속하는 예치 권유는 한국에서 **유사수신행위의 규제에 관한 법률** 위반 소지
- 미래 수익 추정치를 단정적으로 제시하면 **자본시장법상 부당권유** 소지

### 원칙 (카피 가이드)

1. **원금 보장/무손실 단정 금지.** "원금 걱정 없어요"는 *포인트(하우스 머니)* 예치에 한해, 범위를 명확히 표기.
2. 본인 원금 예치 화면에는 **손실 가능성·원금 비보장**을 명확히 고지.
3. 수익률·미래 금액은 **"확정 아님, 변동 가능" 고지 + 과거/현재 기준 명시**.
4. "이자"라는 단어가 예금 이자(보장)로 오인되지 않게 주의 → 실제로는 변동 수익.
5. 법무/컴플라이언스 검토를 원금 예치 기능 출시 전 필수 게이트로 둔다.

> 즉, 온보딩·UX를 쉽게 만드는 것과 **오인 소지 없는 정직한 고지**는 반드시 양립시켜야 함.
