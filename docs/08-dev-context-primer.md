# Naduri — 개발자용 컨텍스트 프라이머

> 최종 업데이트: 2026-07-22 (v10 기준 재정렬)
> **디자인·제품 기준(source of truth) = [prototype/naduri_v10.html](./prototype/naduri_v10.html) + 리포 루트 [CLAUDE.md](../CLAUDE.md).**

## [공통] 프로덕트 한 줄

Naduri = 걷기로 크립토 무경험자를 입장시켜, **걸어서 번 포인트로 디파이 이자를 처음 맛보게** 하고, 그 신뢰로 **원화 예치(→ 기와체인 TVL)** 와 **비트코인·금 소액 매수**까지 데려오는 모바일 앱. 기와체인(GIWA, 두나무/업비트의 OP Stack EVM L2)과 업비트 정산 레일 위에서 돈다.

## [공통] 화폐 모델 (v10)

- **포인트(P)** — 걸어서 버는 적립 화폐. 걸음·출석·퀴즈·게임·영상·보너스·룰렛·뽑기·설문·초대로 쌓인다.
- **원화 자산(원)** — 진짜 돈. 포인트샵에서 포인트를 현금으로 바꾸거나 직접 입금하면 쌓인다. 내 계좌로 출금(영업일 1~3일).
- **비트코인 / 금(PAXG)** — 포인트샵에서 포인트로 매수, 내 지갑에 보관, 원화로 되팔기. 업비트 오라클 시세.
- **이자받기(저금)** — 포인트도 원화도 저금하면 GIWA 디파이에서 이자가 붙는다(매 초 증가). 언제든 원탭으로 꺼낸다.

## [공통] 논리 사슬 (이걸 깨는 설계는 틀린 설계)

1. **걷기 = 리텐션 엔진** (금액이 아니라 매일 리셋되는 습관이 훅)
2. **디파이 = 회수 엔진** (진짜 매출은 rev share; 걷기 보상으론 못 번다)
3. **포인트(하우스 머니) = 둘을 잇는 다리** (첫 저금/전환 문턱이 0)
4. **광고 = 규모까지 버티는 연료** (걷기 보상 원가를 덮음)
5. **포인트 이연 = 런웨이** (테스트넷 동안 원금은 장부 숫자, 현금 유출은 이자만)

## [공통] 규제 방침 (파일럿 단계)

- **디파이 제품이므로, 파일럿/데모 단계에서는 규제(특금법·선불전자지급수단·유사수신·투자권유 등)를 고려하지 않는다.** 포인트↔현금·코인 전환, 지갑·코인·체인 노출을 규제 이유로 막지 않는다.
- 아래 유저 보호 규칙(1~9)은 규제가 아니라 **신뢰·정직·UX** 근거로 지킨다. 신뢰가 유일한 해자다.

## [공통] 불변 규칙 (유저 보호 — 어겨선 안 됨)

1. **유저를 절대 속이지 않는다.** 가짜 잔고, 숨은 차감, 오해 유도 UI 금지.
2. **출금(받기)은 항상 원탭·진짜 열림.** 강요·마찰·"정말 나가실 건가요?"·인질형("출금하려면 입금") 금지. 3탭 이내.
3. **확정 수익률/원금 보장 약속 금지.** 미래 가치·이자는 항상 가정형 + "시장 상황에 따라 달라질 수 있어요". 퍼센트 대신 **붙는 금액**으로 보여준다.
4. **손실 가능성을 감언이설로 숨기지 않는다.** 값이 오르내리는 자산(비트코인·금)엔 "산 금액보다 적어질 수 있어요"를 정직하게 고지(v10 구매 시점처럼). "원금 걱정 없어요"는 포인트(하우스 머니) 저금에 한정. (없는 기능을 넣지 말라는 게 아니라, 원금 잃을 수 있는데 없는 척하지 말라는 뜻.)
5. **체인 복잡성은 대신 처리한다(임베디드 월렛/AA).** 가스·네트워크 선택·스왑 절차를 유저가 하게 만들지 않는다. 단, **지갑주소·비트코인·금·GIWA 체인은 내 지갑·포인트샵·이자받기 맥락에서 드러낸다.** 시드문구·개인키는 "고급 설정"에만.
6. **idle 잔고·위치·의도 데이터는 되비추는 거울 용도만.** 서버 축적·판매 금지.
7. **포인트는 걸어서 버는 적립 화폐이며, 포인트샵에서 현금·비트코인·금·쿠폰으로 전환할 수 있다.**
8. **-EV 갓챠·사행성 뽑기 금지.** 뽑기 경품 기대값이 참가비 이상이 되도록.
9. **토큰 발행은 기본이 아니다.** 요청 없이 토크노믹스·플랫폼 토큰을 제안하지 마라.

## [공통] 지금 단계 (테스트넷 / 파일럿 / gasok 2차)

**지금 만든다**: 로그인, 걷기·적립, 오프체인 포인트 원장, 혜택 미니게임/적립, 포인트→현금/자산 전환, 이자 시뮬레이션(우리 재원), 출금 정산, 내 지갑·포인트샵, 스마트컨트랙트 **"설계"**(문서/다이어그램).

**지금 안 만든다**: 실제 온체인 풀·실거래, 업비트 계정 연동(파트너십 후), 실데이터 idle 개인화(고정 예시), **광고 SDK**(파일럿 검증 오염), 플랫폼 토큰.

**파일럿의 유일한 검증 지표**: 포인트 체험 → **첫 진짜 원금(₩1,000이라도) 전환율**. 설문 의향 아님, 실제 입금 행동으로.

## [FE] 프런트엔드

**스택**: React Native / Expo(SDK 57), TypeScript, React Navigation, `expo-sensors`(만보), `react-native-svg`(링). 상태 로컬 우선, 서버는 포인트 원장·이자·전환·정산.

**앱 구조 (하단 6탭)**: 홈 · 혜택 · 이자받기 · 포인트샵 · 내 지갑 · 내정보. 상세는 [07-user-flow](./07-user-flow.md).

**FE가 반드시 지킬 것**:
- 출금/자산 되팔기 버튼은 **3탭 이내**, 확인 다이얼로그로 막지 말 것.
- 미래 수익률·이자는 **확정형 금지, 항상 가정형 + "달라질 수 있어요"**. 퍼센트 대신 금액.
- 원화·자산 예치 화면엔 **원금 비보장·손실 가능 고지**.
- 가스·시드문구·스왑·네트워크 선택은 노출 금지(임베디드 월렛 뒤). 지갑주소·코인·GIWA는 내 지갑/포인트샵/이자받기에서 노출.
- 40~50 타겟: 큰 글씨·명확한 대비·쉬운 일상어. 적립 화폐 명칭은 **"포인트(P)"**.

## [BE] 백엔드

**핵심 구조**: **오프체인 포인트/자산 원장이 1순위.** 걷기 적립·혜택 적립·저금·이자·전환·출금이 전부 원장 위 트랜잭션(더블엔트리). 온체인은 파일럿에선 설계만.

- **일일 적립 한도**: 200P(걸음·출석·퀴즈·게임·영상). 보너스·룰렛·설문·초대는 한도 제외. **한도는 서버가 권위**.
- **포인트 이연**: 걷기로 번 포인트는 테스트넷 동안 장부 숫자. 실제 지급 재원(rev share/지원금/두나무 지원)을 **"미지급 부채"**로 추적.
- **파일럿 이자**: 저금 이자는 실제 디파이 수익률 반영하되 **파일럿 재원(지원금)으로 지급**, 일회성 마케팅비로 회계 분리. 이자 계산은 원금×경과시간 기반 서버 산출(조회 시 accrue).
- **전환/정산**: 포인트→현금(원화 자산 적립), 포인트→BTC/금(보유 수량 증가), 자산→원화(되팔기), 원화→내 계좌(출금). **모든 금전 뮤테이션은 Idempotency-Key로 멱등**.
- **입금-이자-출금 루프**는 유저가 실제 완주 가능해야 한다(출금 진짜 됨).

**스마트컨트랙트 설계(gasok 제출용, EVM/Solidity, 기와 문서로 검증)**:
- 코어 루프를 컨트랙트로: 포인트/자산 예치 → 스테이블 풀 → 출금. 감사 가능·투명한 보상 풀.
- **풀 선정 보수적으로.** 디페그·컨트랙트 리스크 한 번이 신뢰를 붕괴. 리스크 관리가 생명줄.
- 업비트 정산·idle 연동은 파트너십 전제 인터페이스로 **"설계만"**.

**BE가 반드시 지킬 것**:
- 출금 API는 **한 번의 호출로 완결**, 서버가 마찰·인질형 게이팅을 넣지 않는다.
- 이자/수익 API는 **금액(P/원)으로 응답**, 확정 % 를 계약으로 노출하지 않는다.
- 크립토 지급/커스터디(BTC/금)는 **업비트(라이선스 VASP)** 가 처리하는 구조로 설계. 우리가 직접 환전/커스터디하지 않는다.

**API 스펙**: [11-api-spec](./11-api-spec.md) — FE↔BE 계약 초안(백엔드 미개발, 넘길 문서).
